A manopla do Cientista de Dados

Um cientista e dados também pode ter a sua a sua manopla, levando em consideração o uso da linguagem Python, a mais popular para Data Science nos dias atuais. Com ela em mãos e o seu conhecimento na área de negócios, matemática, estatística e ademais. O portador da manopla seria capaz de trabalhar em grandes empresas como Google. Fonte: Eu mesmo \o/. Mas quais são as joias, agora chamadas de bibliotecas, que compõem a manopla do cientista de dados?


Pandas

A biblioteca Pandas dá ao portador o poder de manipular, preparar e tratar os dados. A biblioteca foi criada com o objetivo de manipular de forma rápida e expressiva, dados estruturados. Ocupando o maior espaço da manopla do cientista de dados, o pandas é um dos componentes principais no portfólio Python para análise de dados e excelente opção para data mining, data wrangling. Uma das fases mais importantes no processo de Data Science é a preparação dos dados, etapa que deve consumir de 50 à 70% do tempo do cientista de dados. Nessa etapa o uso do pandas é essencial e largamente utilizado.

Para trabalhar com Pandas você deve conhecer bem as estruturas Series e DataFrames. Mas o que são Series e DataFrames? Series é um array unidimensional que contém um array de dados e um array de labels, chamado índice. Já os dataframes representam uma estrutura tabular semelhante a estrutura de uma planilha do excel, contendo uma coleção de colunas e registros. O Pandas é a biblioteca mais usada no dia a dia dos cientistas de dados, fazendo dela a biblioteca mais importante da manopla do cientista de dados.

Matplotlib

A biblioteca Matplotlib dá ao portador o poder de visualizar os dados. Existem outras opções de bibliotecas que são capazes de visualizar dados, como Seaborn ou Plotly, entretanto, Matplotlib é a biblioteca do Python mais popular para visualização de dados e geração de gráficos. A biblioteca é essencial para a manopla do cientista de dados, através dela será possível apresentar um resultado ou insight.

O portador tem total controle de estilos de linhas, propriedades de fonte, cores, tamanhos e atributos que deseja inserir em seu gráfico, para assim passar a informação mais clara possível para quem esta visualizando o gráfico. Mas lembre-se, "menos é mais", tome cuidado para não inserir muitos atributos em seu gráfico e dificultar o entendimento de quem está olhando para ele, pela primeira vez e não está familiarizado. Um gráfico deve ser claro e objetivo, para que qualquer pessoa possa entender ao olhar pela primeira vez.

NumPy

A biblioteca NumPY dá ao portador o poder fundamental para computação matemática em Python. A biblioteca fornece suporte para arrays e matrizes, permitindo trabalhar de forma eficiente com vetores e matrizes em python, além de funções matemáticas para operações com esses objetos. O que faz da biblioteca ser tão importante para a manopla do cientista de dados sãos os seus arrays, com alta capacidade de processamento. Um array numpy é um conjunto de valores, todos do mesmo tipo e indexados por uma tupla de valores não negativos. O NumPy ainda adiciona ao Python uma de suas funções principais para análise de dados, a possibilidade de passar dados de um algoritmo para outro.

Mas por que é importante usar arrays com NumPy? Os arrays são homogêneos e possuem tipos estatísticos, oferecem rápida implementação de funções matemáticas e opção mais rápida e mais poderosa que listas. Esses são só alguns exemplos dos benefícios que o uso de arrays com NumPy pode trazer, ainda que a compreensão sobre arrays ajudará a utilizar ferramentas do pandas de forma mais eficaz.

SciPy

A biblioteca SciPy dá ao portador o poder de trabalhar com pacotes que tratam problemas em domínios específicos em computação cientifica. O SciPy é um conjunto de algoritmos matemáticos e funções que permitem estender as funcionalidades do NumPy. A biblioteca NumPy que foi falada anteriormente, na verdade é um subconjunto do SciPy. Assim como na biblioteca Scikit-learn a união com outras bibliotecas permiti trazer grandes resultados. Quando SciPy e NumPy são utilizados juntos, possuem por exemplo as mesmas funcionalidades do Matlab.

A biblioteca SciPy é tão poderosa e de extrema importância para a manopla do cientista de dados que com SciPy uma sessão interativa do Python, torna-se um ambiente de processamento de dados e sistema de prototipagem tal como Matlab, Octave, R-Lab e SciLab.

Scikit-learn

A biblioteca Scikit-learn dá ao portador o poder de trabalhar com Machine Learning em Python. Scikit-learn é a biblioteca mais poderosa e completa para Machine Learning em Python. Diferente da biblioteca pandas, o Scikit-learn é usado para criar modelos e não para a leitura, manipulação e sumarização de dados. Contendo ferramentas simples e eficientes para a mineração de dados e análise de dados. Por outro lado, é possível implementar uma ampla variedade de algoritmos e processos de aprendizado de maquina.

A biblioteca é ainda mais poderosa por fazer uso de outras bibliotecas, NumPy, SciPy e Matplotlib, com essa união o Scikit-learn é capaz de criar ferramentas eficientes para Machine Learning e modelagem estatística como classificação, regressão, clustering e redução de dimensionalidade.

Tensorflow

A biblioteca Tensorflow dá ao portador o poder trabalhar com Machine Learning, Deep Learning e uma ampla variedade de tarefas. O Tensorflow é uma plataforma open source de Machine Learning, a sua principal vantagem comparado a biblioteca Scikit-learn é a ampla variedade de recursos para se trabalhar com Machine Learning. A biblioteca possui um ecossistema flexível e abrangente de ferramentas, bibliotecas e recursos da comunidade que permite que os pesquisadores desenvolvam o que há de mais moderno em Machine Learning, ainda que os desenvolvedores conseguem construir, treinar e implementar facilmente modelos de Machine Learning, utilizando APIs intuitivas de alto nível, como Keras, com execução ágil, o que facilita a iteração imediata do modelo e a depuração fácil.

A biblioteca Tensorflow é muito semelhante ao NumPy, por ambos serem bibliotecas de matriz N-dimensional, entretanto, diferente do Tensorflow o NumPy tem suporte a n-dimensão, mas não oferece métodos para criar funções de tensor e automaticamente computar derivas, além de não ter suporte a GPU. Por isso surgiu o Tensorflow.
